# 🌉 Deploy Bridge Setup Guide

Complete guide to set up automatic deployment from GitHub to free hosting platforms.

## 🚀 Quick Setup (5 Steps)

### Step 1: Choose Your Platform

| Platform | Best For | Free Tier |
|----------|----------|-----------|
| **Vercel** | Next.js, React | 100GB bandwidth |
| **Netlify** | Static sites | 100GB bandwidth |
| **GitHub Pages** | Simple static | 1GB storage |
| **Cloudflare Pages** | Global CDN | Unlimited builds |
| **Fly.io** | Docker containers | 3 VMs |
| **Render** | Web services + DB | 750hrs + PostgreSQL |

### Step 2: Add GitHub Secrets

Go to `Settings → Secrets → Actions` and add:

#### For Vercel
```
VERCEL_TOKEN       = your-token
VERCEL_ORG_ID      = your-org-id
VERCEL_PROJECT_ID  = your-project-id
```

#### For Netlify
```
NETLIFY_AUTH_TOKEN = your-token
NETLIFY_SITE_ID    = your-site-id
```

#### For Render
```
RENDER_API_KEY      = your-api-key
RENDER_BLUEPRINT_ID = your-blueprint-id
```

#### For Cloudflare
```
CLOUDFLARE_API_TOKEN      = your-token
CLOUDFLARE_ACCOUNT_ID     = your-account-id
CLOUDFLARE_PROJECT_NAME   = your-project
```

#### For Fly.io
```
FLY_API_TOKEN = your-token
FLY_APP_NAME  = your-app-name
```

#### For GitHub Pages
```
(No secrets required!)
```

### Step 3: Copy Workflow to Your Repo

```bash
# Copy specific platform workflow
cp examples/vercel/vercel.yml .github/workflows/deploy.yml

# OR copy the multi-platform workflow
cp .github/workflows/deploy-all.yml .github/workflows/deploy.yml
```

### Step 4: Configure Your Build

Update the workflow to match your project:

```yaml
- name: Install Dependencies
  run: npm ci

- name: Build
  run: npm run build  # Change to your build command

- name: Upload Artifact
  uses: actions/upload-pages-artifact@v3
  with:
    path: ./dist  # Change to your output directory
```

### Step 5: Push and Deploy! 🚀

```bash
git add .
git commit -m "Add auto-deploy"
git push
```

## 🔧 Platform-Specific Setup

### Vercel

1. Get token: https://vercel.com/account/tokens
2. Get IDs: `npm i -g vercel && vercel projects list`
3. Add to GitHub secrets
4. Push!

### Netlify

1. Get token: https://app.netlify.com/user/applications
2. Get site ID from site settings
3. Add to GitHub secrets
4. Push!

### GitHub Pages

1. Go to Settings → Pages
2. Source: GitHub Actions
3. No secrets needed!
4. Push!

### Cloudflare Pages

1. Get API token: https://dash.cloudflare.com/profile/api-tokens
2. Get Account ID: Workers & Pages → Overview
3. Create project: `wrangler pages project create my-project`
4. Add secrets
5. Push!

### Fly.io

1. Install: `curl -L https://fly.io/install.sh | sh`
2. Login: `flyctl auth login`
3. Create app: `flyctl apps create my-app`
4. Deploy once locally: `flyctl deploy`
5. Get token: `flyctl auth token`
6. Add secrets
7. Push!

### Render

1. Get API key: https://dashboard.render.com/settings/api-keys
2. Create Blueprint: `render blueprint create`
3. Get Blueprint ID
4. Add secrets
5. Push!

## 🐳 Docker + Fly.io (Recommended for Full-Stack)

### 1. Create Dockerfile

```dockerfile
FROM node:20-alpine
WORKDIR /app
COPY package*.json ./
RUN npm ci --only=production
COPY . .
EXPOSE 8080
CMD ["node", "index.js"]
```

### 2. Create fly.toml (generated by Fly.io)

```toml
app = "my-app"
primary_region = "iad"

[build]
  dockerfile = "Dockerfile"

[[http_service]]
  internal_port = 8080
  force_https = true
  auto_stop = false
  auto_start = true

[[statics]]
  guest_path = "/app/public"
  url_prefix = "/"
```

### 3. Deploy

```bash
flyctl deploy
flyctl secrets set DATABASE_URL=postgres://...
```

## 🔄 Updating Your Deployment

### Update Workflow

Just edit `.github/workflows/deploy.yml` and push!

### Update Secrets

1. Go to Settings → Secrets → Actions
2. Click "Update" next to the secret
3. Enter new value

### Rollback

1. Go to Actions tab
2. Find the last successful deployment
3. Click "Re-run all jobs"

## 🆘 Troubleshooting

### "Workflow not found"
- Make sure workflow is in `.github/workflows/`

### "Secret not found"
- Check secrets are in Settings → Secrets → Actions
- Verify secret name matches workflow exactly (case-sensitive)

### "Build failed"
- Check package.json scripts match workflow
- Verify Node.js version in workflow matches local

### "Deploy succeeded but site not updated"
- Check platform dashboard for deployment status
- Verify branch is main (or configured branch)
- Check for caching issues

### "Permission denied"
- For GitHub Pages: Check Actions has read/write permissions
- For SSH: Verify private key is correctly added
